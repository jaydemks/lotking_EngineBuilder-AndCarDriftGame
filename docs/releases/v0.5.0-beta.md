# Published: 2026-07-04 21:17:28 +0200 (tag v0.5.0-beta)


**Build It. Drift It. Drive It.**

First Git-ready beta baseline for Lot King Engine Builder: a browser drift game with its own in-game engine editor.

You can play drift tracks, open the Engine Editor, build or modify levels, tune the player vehicle, manage assets, design engine sounds, edit HUD/radio layout, save projects locally, export/import editor projects, and package playable ZIP builds.

This release is not the final stable version. It is a recovery and consolidation milestone: the runtime has been split into focused modules, the editor has been partially refactored, and several recent editor regressions were restored after a risky extraction. More cleanup, testing, and editor hardening will continue before the future stable release.

## Highlights

- Browser-based drift game with no build step.
- Engine Editor with scene outliner, inspector, transform gizmos, snapping, undo/redo, and viewport camera controls.
- Local multi-level workflow: new, save, save-as, load, import, export, duplicate, rename, delete.
- Asset browser for levels, GLB/GLTF imports, player blueprints, sound sets, lights, FX, textures, and scene assets.
- Player Blueprint system for vehicle tuning, camera, lights, exhaust, data widgets, spawn, model, and engine sound.
- Engine Sound Designer with RPM loop banks, throttle/off-throttle layers, limiter, turbo, blow-off, backfire, shift, ignition, filters, reverb, and live testing.
- HUD / Radio editor for positioning the frame, screen, buttons, volume/bass controls, and visual layout.
- Playable ZIP export flow for packaging a playable level build.
- Runtime split into focused modules under `js/runtime/`.
- Editor systems split into focused modules under `js/editor/`.

## Beta Notes / Known Issues

This is still a beta baseline.

- The Engine Editor refactor is ongoing and must continue in small verified steps.
- `js/editor/editor.js` is still large and needs more modular extraction.
- Some editor workflows may still need regression testing after the recent recovery pass.
- Large assets such as `models/player.glb` and Blender source files are not included in normal Git and should move through Git LFS or release assets.
- Imported GLB child hierarchy editing and full mesh collision are planned but not finished.
- Playable export, editor project export, and future publish/share flows still need clearer separation.
- Versioned level migrations are planned so older tracks remain compatible as the engine evolves.

## Things I Already Know I Want To Improve

This beta is a foundation, not the finish line. A lot of the next work is already clear, even if the full public roadmap is still in my head and will evolve as the project stabilizes.

- Finish the Engine Editor refactor so large files like `js/editor/editor.js` become smaller, safer modules.
- Keep hardening the editor after the recent recovery pass, with more regression checks before each refactor slice.
- Improve vehicle asset preparation. I plan to release a small Blender helper/plugin for faster vehicle rigging. The first version will target exactly four-wheel vehicles and a simple structure per wheel: wheel, brake disc, and caliper.
- Add better controller support where possible, including USB/gamepad input for driving and editor testing.
- Expand material editing and material management for imported GLB/GLTF assets.
- Improve imported model workflows, including hierarchy editing, replacement, collision setup, and reusable asset metadata.
- Separate more clearly the flows for editor project export, playable ZIP build, and future sharing/publishing.
- Add versioned level migrations so old tracks keep loading as the engine changes.

This release is mainly a foundation: the project is now named, licensed, documented, tagged, and ready to keep evolving toward the future stable beta.
