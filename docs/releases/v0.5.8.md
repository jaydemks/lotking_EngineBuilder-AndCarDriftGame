# LOT KING ENGINE EDITOR v0.5.8 - Complex colliders + video experiments

## v0.5.8 - Complex colliders + video experiments

### Release Status

- Status: released.
- Tag target: `v0.5.8`.
- Scope: complex collider authoring for GLB child meshes, selectable collider dummies in the Scene outliner, per-dummy collision mode overrides, experimental video settings, screen-space volumetric lighting, and built-in Pillar collider cleanup.

### Added

- Complex GLB colliders now expose their generated child collider dummies under the parent object in the Scene outliner.
- Complex collider dummy rows are selectable individually from the Scene list.
- Selecting a complex collider dummy opens per-dummy controls in the Physics inspector.
- Per-dummy collider mode can now be set to:
  - `Complex mesh`: keeps following the auto-generated GLB child mesh box.
  - `Solid box`: stores manual collider edits for that dummy.
  - `Off`: disables only that collider part.
- Transform gizmos now work on the selected complex collider dummy instead of only the parent collider.
- Complex collider helpers now visually distinguish the selected dummy and manually solidified dummy boxes.
- Added Video settings entries in both editor and game menus:
  - Renderer: `Three.js WebGL` / `Raytracing experimental`.
  - Volumetric lighting: on/off.
- Added a WebGL-compatible screen-space volumetric lighting pass in the runtime post-processing stack.
- Video settings state is now carried through the shared runtime settings module and exposed on the renderer metadata.

### Improved

- The Physics inspector now explains the difference between the default solid collider and complex GLB child collider mode.
- Complex collider outliner rows are kept under their owning scene object, preserving the existing parent/child Scene layout.
- Complex collider part edits are stored in `colliderShape.parts`, so saved/imported scene objects can keep per-part overrides.
- Raytracing selection is now represented as an explicit experimental preset with safe WebGL fallback instead of silently implying full renderer replacement.

### Fixed

- Built-in `Pillar` objects now use box colliders instead of circle/cylindrical colliders, matching their square pillar mesh and base.
- Switching an object between normal object selection, root collider selection, player collider selection and complex dummy selection now clears stale collider selection state.
- Complex collider helper highlighting now follows the selected dummy row.

### Notes / limitations

- `Raytracing experimental` currently remains a guarded preset/fallback. The referenced raytracing renderer is not a drop-in replacement for the full arbitrary Three.js scene used by the editor/game.
- Volumetric lighting is implemented as a lightweight WebGL screen-space pass for this release. A full WebGPU/froxel volumetric lighting system remains a future rendering milestone.
- Complex collision is still box-based per child mesh, not true triangle mesh physics. This release focuses on authoring and per-part control without destabilizing the current arcade physics layer.
- Light Pole and Cone objects intentionally keep circle/cylindrical colliders because their visible shapes are round or narrow.

### Deferred

- Cinema Studio scene element and sequencer workflow.
- Deep material editing pass for scene elements.
- True projected decal geometry beyond the current Free Texture / Decal workflow.
- Full raytraced renderer integration for arbitrary saved editor scenes.
